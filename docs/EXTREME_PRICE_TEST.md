# 极端价格场景测试

目标：验证价格靠近 0/1 时，系统不会产生非法报价或异常下单。

## 1. 代码侧保护

- 策略侧钳制：`safe_price()` 将报价限制在 `[tick, 1-tick]`（默认 `[0.01, 0.99]`）
- 发单前对齐：价格按 tick 向下取整；数量按 2 位小数向下取整
- size 若取整后 `< 0.01`，本地直接拒绝发送
- 交易所返回最小 tick 拒单时，执行层可运行时学习 tick 并重试一次

## 2. 建议测试场景

1. YES 接近 0.99 / NO 接近 0.01
2. YES 接近 0.01 / NO 接近 0.99
3. 快速单边波动 + 高频重报价

## 3. 观察点

- 无负数价格
- 无 `> 1` 价格
- 精度拒单（tick/size）出现时可被正确分类并冷却
- OFI 毒性触发后能及时撤单
- 极低名义金额下，若触发 `min size:$1`，不会出现毫秒级重复下单风暴

## 4. 运行方式

```bash
PM_DRY_RUN=true cargo run --bin polymarket_v2
```

有条件时再小仓 live 复验。
