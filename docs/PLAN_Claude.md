     │ 2. 残仓为什么总是输？——逆向选择现象                                                                                                                          │
     │                                                                                                                                                              │
     │ April 10 三个 session 的残仓都在输，这不是偶然：                                                                                                             │
     │                                                                                                                                                              │
     │ 机制分析：                                                                                                                                                   │
     │ 在一个趋势性市场（比如 NO 概率从 0.5 升到 0.9）：                                                                                                            │
     │ 1. Smart money taker 不断卖 YES 给我们（YES 价格下跌中）                                                                                                     │
     │ 2. 我们的 YES maker 不断被填 → 积累 YES 仓位                                                                                                                 │
     │ 3. 想买 NO 来配对，但 NO 已贵 → VWAP ceiling 挡住                                                                                                            │
     │ 4. 结果：持有大量 YES 残仓，市场最终 NO 胜 → YES = 0                                                                                                         │
     │                                                                                                                                                              │
     │ 反向亦然：YES 大涨时，我们的 NO bids 被大量填，积累 NO 残仓，最终 NO = 0。                                                                                   │
     │                                                                                                                                                              │
     │ 这是 maker 在单向趋势市场中的结构性问题，不仅仅是 bug。                                                                                                      │

     │ 4. 策略真正需要解决的问题：残仓尺寸 vs 配对利润                                                                                                              │
     │                                                                                                                                                              │
     │ 数学框架：                                                                                                                                                   │
     │                                                                                                                                                              │
     │ 策略总 EV = EV(paired) + EV(residual)                                                                                                                        │
     │                                                                                                                                                              │
     │ EV(paired) = N_pairs × (1.0 - pair_cost) > 0  ← 已实现，约 +0.05-0.23/对                                                                                     │
     │ EV(residual) ≈ residual_qty × (p_market - avg_cost) ≈ 0  ← 在效率市场中近似为零                                                                              │
     │                                                                                                                                                              │
     │ 但 Var(residual) 很大，在当前配置下 Var(residual) >> EV(paired)                                                                                              │
     │                                                                                                                                                              │
     │ 所以策略并不是"负期望"，而是：                                                                                                                               │
     │ - 期望值本身可能为正或接近零                                                                                                                                 │
     │ - 但方差极大，少数残仓亏损事件可以轻易覆盖多轮配对利润                                                                                                       │
     │                                                                                                                                                              │
     │ 当前参数下残仓风险量化：                                                                                                                                     │
     │                                                                                                                                                              │
     │ max_net_diff = 15 (PM_MAX_NET_DIFF=15)                                                                                                                       │
     │ bid_size = 5                                                                                                                                                 │
     │ max_residual = 15 shares × avg_cost 0.45 = 6.75 USDC 最大残仓价值                                                                                            │
     │ 典型配对利润 = 5-10对 × 0.05/对 = 0.25-0.50 USDC                                                                                                             │
     │                                                                                                                                                              │
     │ 比值：残仓风险 / 配对利润 = 6.75 / 0.38 ≈ 18x                                                                                                                │
     │                                                                                                                                                              │
     │ 只有在以下情况下残仓才能被"抵消"：                                                                                                                           │
     │ 1. 残仓 50/50 押对方向（即 EV(residual) > 0）                                                                                                                │
     │ 2. 或者配对数量足够多（pair_cost 极低，如 0.70，则单对利润 0.30）                                                                                            │
     │                                                                                                                                                              │

     │                                                                                                                                                              │
     │ 路径 A：减小残仓敞口（推荐短期行动）                                                                                                                         │

     │ 目标：残仓最大亏损 << 典型配对利润                                                                                                                           │
     │ 条件：residual_qty × avg_cost < N_pairs × (1.0 - pair_cost)                                                                                                  │
     │                                                                                                                                                              │
     │ 当前 pair_cost ≈ 0.80-0.90 → 每对利润 0.10-0.20                                                                                                              │
     │ 若 N_pairs = 10 → 总配对利润 1.0-2.0 USDC                                                                                                                    │
     │ 允许残仓损失 < 1.0-2.0 USDC → 残仓 ≤ 2-4 份                                                                                                                  │
     │                                                                                                                                                              │
     │ 结论：max_net_diff 需要大幅降低到 ≤ 5                                                                                                                        │

     │                                                                                                                                                              │

     │ VWAP Ceiling 优化计划（新增）                                                                                                                                │
     │                                                                                                                                                              │
     │ Context                                                                                                                                                      │
     │                                                                                                                                                              │
     │ 当前增量 VWAP ceiling 公式允许"消费"早期便宜成交积累的优势，导致 pair_cost 随时间趋近 pair_target，无法在 pair_target 以下锁定稳定利润。本计划通过引入显式   │
     │ pair_cost_safety_margin 来修复这一策略层缺陷。                                                                                                               │
     │                                                                                                                                                              │
     │ ---                                                                                                                                                          │
     │ 问题分析                                                                                                                                                     │
     │                                                                                                                                                              │
     │ 当前公式（pair_arb.rs:268-287）                                                                                                                              │
     │                                                                                                                                                              │
     │ ceiling = [(pair_target - opp_avg) × (same_qty + bid_size) - same_qty × same_avg] / bid_size                                                                 │
     │                                                                                                                                                              │
     │ 化简 = legacy + same_qty × (legacy - same_avg) / bid_size                                                                                                    │
     │ 其中 legacy = pair_target - opp_avg                                                                                                                          │
     │                                                                                                                                                              │
     │ 问题本质                                                                                                                                                     │
     │                                                                                                                                                              │
     │ 当 same_avg < legacy 时（早期低价成交），公式允许后续出价 高于 legacy，等同于把早期盈余"花掉"：                                                              │
     │                                                                                                                                                              │
     │ 示例：pair_target=0.97, NO_avg=0.50 → legacy=0.47                                                                                                            │
     │ YES_avg=0.40 (5份) → ceiling = 0.47 + 5×(0.07)/5 = 0.54                                                                                                      │
     │                                                                                                                                                              │
     │ YES@0.54 成交后 → YES_avg=(0.40+0.54)/2=0.47 → pair_cost=0.97                                                                                                │
     │                                                                                                                                                              │
     │ 早期 YES@0.40 的 7 分优势被完全消耗，最终 pair_cost = pair_target，零利润。                                                                                  │
     │                                                                                                                                                              │
     │ 与 Gabagool/0x8dxd 策略的对比                                                                                                                                │
     │                                                                                                                                                              │
     │ Gabagool 的盈利来源不在 ceiling 本身，而在：                                                                                                                 │
     │ 1. Maker 成交时市价暂时偏离带来的系统性折扣                                                                                                                  │
     │ 2. 双边异步成交（两腿各自捡到低价）                                                                                                                          │
     │ 3. A-S skew 主动压低报价                                                                                                                                     │
     │                                                                                                                                                              │
     │ ceiling 应该是"锁利润的护栏"，而不是"可以花掉的预算"。                                                                                                       │
     │                                                                                                                                                              │
     │ Legacy 公式（pair_target - opp_avg）更保守：早期便宜成交 = 利润锁定，不被后续高价稀释。                                                                      │
     │                                                                                                                                                              │
     │ ---                                                                                                                                                          │
     │ 推荐方案                                                                                                                                                     │
     │                                                                                                                                                              │
     │ 方案 C：增量公式 + 显式安全边距（推荐）                                                                                                                      │
     │                                                                                                                                                              │
     │ // pair_arb.rs::vwap_ceiling                                                                                                                                 │
     │ pub(crate) fn vwap_ceiling(                                                                                                                                  │
     │     pair_target: f64,                                                                                                                                        │
     │     safety_margin: f64,   // 新增参数                                                                                                                        │
     │     opp_avg: f64,                                                                                                                                            │
     │     same_qty: f64,                                                                                                                                           │
     │     same_avg: f64,                                                                                                                                           │
     │     bid_size: f64,                                                                                                                                           │
     │ ) -> f64 {                                                                                                                                                   │
     │     let effective_target = pair_target - safety_margin;  // 新增                                                                                             │
     │     let legacy = effective_target - opp_avg;                                                                                                                 │
     │     if same_qty <= FLOAT_EPS || bid_size <= FLOAT_EPS {                                                                                                      │
     │         return legacy;                                                                                                                                       │
     │     }                                                                                                                                                        │
     │     let numerator = (effective_target - opp_avg) * (same_qty + bid_size) - same_qty * same_avg;                                                              │
     │     let ceiling = numerator / bid_size;                                                                                                                      │
     │     if ceiling.is_finite() { ceiling } else { legacy }                                                                                                       │
     │ }                                                                                                                                                            │
     │                                                                                                                                                              │
     │ 效果：所有 ceiling 计算以 pair_target - safety_margin 为基准，确保 pair_cost ≤ pair_target - safety_margin 始终成立。                                        │
