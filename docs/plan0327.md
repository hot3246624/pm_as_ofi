# GLFT V2 统一对齐控制方案

## Summary
当前主矛盾已经不是 `OFI`，也不是早期的 `forced_realign`。最新日志说明：

- `OFI` 已基本收敛为 `heat`，`toxicity/kill=0`
- `forced_realign` 已明显下降
- 剩下的系统性问题是：**信号层、策略层、发布层都在各自处理“对齐”**，形成重复控制回路，导致 `hard misalignment / pause / recover / drift_prealign` 仍会相互放大

这版方案的目标不是再调阈值，而是做一次**单一对齐控制器收口**：

- 信号层负责定义“当前市场参考中枢是否可信、偏离是否严重、当前允许什么报价状态”
- 策略层只围绕这个统一参考中枢报价
- 发布层只负责“稀疏、安全地把目标发出去”，不再二次发明自己的漂移控制逻辑

## Key Changes

### 1. 信号层：把 `synthetic_mid` 从“硬真值”降级为“有置信度的参考来源”
在 `/Users/hot/web3Scientist/pm_as_ofi/src/polymarket/glft.rs` 重构对齐语义，新增统一参考中枢：

- `synthetic_mid_raw`
- `reference_mid`
- `reference_confidence`
- `reference_drift_ticks`
- `alignment_state`

固定定义：

- `reference_mid = lerp(modeled_mid, synthetic_mid_raw, reference_confidence)`
- `reference_drift_ticks = |modeled_mid - reference_mid| / tick`

`reference_confidence` 固定由以下因子计算，并 clamp 到 `[0,1]`：

- `freshness_conf`
  - Poly book stale `<=1s` 为 `1.0`
  - `>=3s` 为 `0.0`
  - 中间线性插值
- `two_sided_conf`
  - YES/NO 两侧 mid 都可得：`1.0`
  - 只有一侧可得：`0.5`
  - 两侧都不可得：`0.0`
- `edge_conf`
  - `edge_dist = min(synthetic_mid_raw, 1 - synthetic_mid_raw)`
  - `edge_dist <= 3*tick` 时为 `0.0`
  - `edge_dist >= 15*tick` 时为 `1.0`
  - 中间线性插值
- `endgame_conf`
  - 剩余时间 `>60s`：`1.0`
  - `30s~60s`：`0.75`
  - `<=30s`：`0.5`

合成方式固定为乘积：
- `reference_confidence = freshness_conf * two_sided_conf * edge_conf * endgame_conf`

这样处理后：
- 盘口极端、单边、临近收盘、薄书时，`synthetic_mid_raw` 仍可观测，但不会继续被当成硬真值
- 真正进入 `pause` 的前提不再是“raw synthetic 很极端”，而是“**高置信参考** 仍显示模型严重失配”

### 2. 对齐状态机：只保留一套，且由 `reference_drift_ticks + reference_confidence` 驱动
保留 `DriftMode`，但语义改为统一 `alignment_state`：

- `Normal`
  - `reference_drift_ticks < 6`
- `Damped`
  - `reference_drift_ticks >= 6`
- `Frozen`
  - `reference_drift_ticks >= 10`
  - 或 `reference_confidence < 0.75` 且 `reference_drift_ticks >= 8`
- `Paused`
  - 仅当 `reference_confidence >= 0.75` 且 `reference_drift_ticks >= 12`

恢复条件固定：
- `Paused -> Frozen/Damped`
  - `reference_drift_ticks < 8`
  - 且连续健康窗口 `>= 3s`
- `Frozen -> Damped/Normal`
  - `reference_drift_ticks < 6`
- `Damped -> Normal`
  - `reference_drift_ticks < 5`

关键规则固定：
- **低置信参考只允许进入 `Frozen`，不允许直接 `Paused`**
- `Paused` 只代表“高置信度下模型真的错了”，不是“盘口本身已经不可信”

这一步是本轮架构收口的核心。

### 3. 策略层：`glft_mm` 只围绕 `reference_mid` 报价，不再自己解释漂移
在 `/Users/hot/web3Scientist/pm_as_ofi/src/polymarket/strategy/glft_mm.rs`：

- 用 `reference_mid` 替代当前的 `trusted_mid` 语义
- `glft_mm` 不再自行推断“现在是不是该 prealign / hard-pause”
- 仍保留：
  - inventory shift
  - half spread
  - dominant-side penalty
  - dominant-side suppress
- 但这些行为全部以 `alignment_state` 为条件：

固定规则：
- `Normal`
  - 维持当前 GLFT 输出
- `Damped`
  - `half_spread *= 1.15`
  - dominant-side penalty 正常生效
- `Frozen`
  - `half_spread *= 1.35`
  - suppress dominant-side `BUY`
  - 保留便宜侧 `BUY` 与合理 `SELL`
- `Paused`
  - 四槽位静默

这样后：
- 强趋势下不会继续一路追主导侧
- 但也不会因为薄书极端值而过早全停
- 策略层重新回到“只负责报价数学”，不再承担对齐控制职责

### 4. 发布层：`shadow -> publish` 保留，但取消本地漂移治理
在 `/Users/hot/web3Scientist/pm_as_ofi/src/polymarket/coordinator_order_io.rs`：

保留 `shadow_target / published_target` 两阶段发布，但把“对齐控制”从这里抽走。

固定变更：

- **停止为 `glft_mm` 发出 `DriftPrealign`**
- 发布层不再根据本地 `trusted_mid/normalized_target` 自己决定“是否需要 drift prealign”
- 发布层只接受来自信号层的统一结果：
  - 当前 `alignment_state`
  - 当前 `reference_mid`
  - 当前 `normalized_target`
  - 当前 `quote_envelope`

发布原因只保留：
- `InitialDwell`
- `ShadowDwell`
- `UnsafeQuote`
- `CrossRejectRecovery`
- `ForcedRealign`

其中 `ForcedRealign` 的语义收窄为：
- 当前 live quote 已经落到 `quote_envelope` 外
- 且 quote age 超过最小年龄阈值
- 这是执行安全纠偏，不是第二套漂移控制器

固定 dwell：
- `Normal`: `700ms`
- `Damped`: `500ms`
- `Frozen`: `350ms`
- `Paused`: 不发布

固定 misaligned 判定：
- 仍用“相对失真”而非绝对价格
- 但只相对 `reference_mid + normalized_target`
- 不再相对独立的 publish-layer 漂移阈值

### 5. Warm-start 与 ready：保持现状，但纳入统一参考置信度
这轮不再重做 startup pipeline，只做一条收口：

- `warm-start accepted / soft-accepted / rejected` 保持现有逻辑
- 但 warm-start 只影响：
  - `basis/sigma/A/k` 初值
- **不绕开 `reference_confidence`**
- `soft-accepted` 期间，`alignment_state` 上限固定为 `Damped`
  - 直到首个 live fit ready 后解除
- `rejected + last_good_fit` 仍允许 ready，但同样受 `reference_confidence` 约束

这样可以保住当前 `2~3s` ready 收敛，同时避免 warm-start 把对齐状态带偏。

### 6. 观测与统计：把“对齐控制”做成可验证单源
升级日志与 shutdown 统计，确保下一轮可以直接验证方案是否成功。

新增日志字段：
- `synthetic_mid_raw`
- `reference_mid`
- `reference_confidence`
- `reference_drift_ticks`
- `alignment_state`
- `pause_reason`
- `dominant_side`
- `dominant_buy_penalty_ticks`
- `publish_reason`
- `distance_to_reference_ticks`
- `distance_to_normalized_target_ticks`

shutdown 统计固定增加：
- `alignment_pause_events`
- `alignment_frozen_ticks`
- `alignment_damped_ticks`
- `publish_shadow_dwell`
- `publish_unsafe_quote`
- `publish_forced_realign`

要求：
- 下一轮日志里，能直接分辨是“参考置信度下降导致 Frozen”，还是“高置信严重偏离导致 Paused”
- 不再需要靠多个局部日志拼接推断

## Test Plan

### 1. 信号层
- 双边健康盘口、reference 高置信、`modeled_mid` 偏离大于 `12 ticks`：
  - 进入 `Paused`
- 单边或极端盘口、reference 低置信、同样偏离：
  - 最多进入 `Frozen`
  - 不允许误进 `Paused`
- `reference_confidence` 在 stale / 单边 / 边缘价格 / 临近收盘时正确下降

### 2. 策略层
- `Normal` 下行为与当前版本基本一致
- `Damped` 下只 widen，不静默
- `Frozen` 下 suppress 主导侧 `BUY`，保留便宜侧
- `Paused` 下四槽位静默
- 不再出现“引用 raw synthetic 极端值直接把买单推到不合理中枢”的行为

### 3. 发布层
- `glft_mm` 不再发出 `DriftPrealign`
- 连续小漂移仍被 `shadow_dwell` 吸收
- `ForcedRealign` 仅在 quote 脱离 envelope 时触发
- 相对市场合理的低价单可以存在；但相对 `reference_mid` 严重失真的旧单必须被纠正

### 4. 回放验收
针对当前这类 `2026-03-27` 日志形态，验收目标固定为：

- `toxicity/kill` 继续维持接近 `0`
- `forced_realign(total)` 不高于当前版本
- `hard basis misalignment` 次数显著少于当前版本
- 不再出现一轮里多次 `pause -> recover -> 再pause` 的短周期往返
- 不再出现因对齐控制抖动导致 `placed` 突降到接近停摆的轮次

## Assumptions
- 主线继续是 `PM_STRATEGY=glft_mm`
- 本轮不改 `OMS / Executor` 接口，不改 slot-keyed 架构
- 本轮不再改 `OFI` 数学；它已经不是当前主矛盾
- 不新增 env 参数；阈值先固化为内部常量
- 保留 `0-1` 全区间报价能力；修的是“参考中枢与对齐控制”，不是做绝对价格钳制
