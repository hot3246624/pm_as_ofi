## 2. 先说结论：`xuan` 属于哪一类量化交易

如果只给一个最贴切的标签，我会把 `xuan` 归到：

**`alpha-aware inventory-based market making`**

中文可以翻成：

**带轻方向偏置的库存管理型微观结构做市**

这句话要拆开理解。

### 2.1 `market making`

他不是主要靠押单边趋势赚钱。

他的核心问题不是：

- “涨还是跌”

而是：

- “我什么时候值得开第一腿”
- “开了之后能不能尽快补到 opposite”
- “如果补不到，怎么把库存风险压住”

这就是做市思维。

### 2.2 `inventory-based`

他真正管理的核心对象不是信号，而是库存。

他每开一条 first leg，实际上都在制造一个风险状态：

- 有一边先成交了
- 另一边还没成交
- 这段时间里，他就带着单边风险

所以他的真正 edge，很大一部分不在“看对了方向”，而在：

- 什么时候开风险
- 开多少风险
- 风险开出来后如何快速关掉

### 2.3 `alpha-aware`

他又不是纯中性机械做市。

当前研究已经很明确：

- `MERGE` 前存在 `winner-bias`
- 第一腿有弱 winner-side 倾向
- 更高价侧和更强一侧往往更偏 winner-overweight

这意味着他不是无脑双边买、无脑配对。

他明显在做 **状态筛选**。

所以最准确的理解不是：

- “他是套利员”

而是：

- “他是会挑状态开仓的库存管理型做市者”

## 4. 第一章：市场微观结构

这是你必须先真正理解的一章。

如果你不懂微观结构，你会永远把很多问题误解成：

- “是不是 pair target 设错了”
- “是不是价格阈值不对”

实际上很多问题根本不是价格公式问题，而是：

- 盘口结构问题
- 成交节奏问题
- 队列与优先级问题

### 4.1 什么是 `bid` / `ask`

最简单地说：

- `bid`：别人愿意买的最高价
- `ask`：别人愿意卖的最低价

两者之间的差：

- `spread`

如果你主动去打 `ask`，你通常更快成交，但更贵。
如果你挂在 `bid` 等别人来打你，你通常更便宜，但不一定成交。

这就是：

- `taker`
- `maker`

### 4.2 什么是 `maker` / `taker`

#### `taker`

你主动吃掉当前盘口上的单。

特点：

- 更快
- 更确定
- 成本更高

#### `maker`

你把单挂在盘口上，等别人来成交你。

特点：

- 更便宜
- 但不保证成交
- 容易被挑单

你要记住一句最重要的话：

> 做市策略最大的敌人，不是“价格没到”，而是“只成交了你最不想先成交的那一边”。

### 4.3 什么是 `adverse selection`

这在你们项目里极其重要。

意思是：

- 你挂着单
- 对手只在对自己有利的时候来打你

结果就是：

- 你以为自己在“低成本成交”
- 实际上你是在“只在最糟的时刻先成交”

这会直接制造：

- first leg 成交后 opposite 难补
- 库存越积越危险

这也是为什么 `xuan` 的强大之处，未必是他单纯会挂单，而是：

- 他似乎知道什么状态下挂出去，更容易既成交、又不被严重 adverse selection

### 4.4 什么是 `fill probability`

这就是：

- 你挂一个单后，在给定时间内成交的概率

对你们当前问题来说，它至少要分成两层：

1. `first-leg fill probability`
2. `opposite fill probability`

很多人只关心第一层，这是错的。

因为你们真正关心的不是：

- “第一腿能不能成”

而是：

- “第一腿成了之后，另一腿能不能在 30s 内成”

这才是 `xuan` 的核心问题。

### 4.5 什么是 `queue` / 排队位置

在挂单时，价格一样的人要排队。

越靠前：

- 越容易先成交

越靠后：

- 越不容易成交

这意味着一个很重要的现实：

即使你看见了同样的盘口、挂了同样的价，你也不一定能复制别人的 fill 结果。

所以公开数据研究只能回答：

- “什么状态看起来更容易成功”

但它不能完全回答：

- “你自己能不能以同样方式成功”

后者必须靠你们自己的执行真值。

### 4.6 这一章学完，你要得到什么能力

你至少要能解释这 3 件事：

1. 为什么 `maker` 不等于“更安全”
2. 为什么 first leg 成交本身可能就是风险开始
3. 为什么 `xuan` 的关键不只是会挂单，而是会挑状态挂单

## 5. 第二章：做市与库存管理

这是理解 `xuan` 的第二核心。

### 5.1 什么是库存风险

假设你想做一对：

- 先买 `YES`
- 再买 `NO`

如果两边都买到，你接近一套完整对冲。

但如果只买到第一边，你就带了单边风险。

这就叫：

- `inventory risk`

在传统做市里，库存风险是：

- 你买太多现货或卖太多现货

在你们这里，库存风险更具体：

- 你只成交了一边 binary leg
- 还没补上另一边

### 5.2 为什么“继续挂买单压低成本”不总是对的

你前面已经敏锐地指出过这个问题。

你们旧处理方式更像：

- first leg 成了
- opposite 没成
- 那就继续同侧挂、更便宜地挂、摊薄成本

这种做法的问题是：

- 你可能在最不该扩大风险的时候继续扩大风险

而 `xuan` 看起来更像：

- 如果这轮配对没按预期完成
- 到一定亏损 / 超时 / 状态变化，就直接 repair 或 aggressive completion

这说明他更重视：

- 风险关闭

而不是：

- 成本看起来更漂亮

### 5.3 什么是 `completion-first`

这不是一个口号，而是一种风险哲学：

> 一旦第一腿成交，后续目标不再是“寻找更便宜的新机会”，而是“尽快把当前 episode 关掉”。

这会带来 3 个直接后果：

1. 不再轻易做同侧 risk-increasing add
2. 会更关注 opposite fill 的可达性
3. 会有更清晰的超时 / repair 边界

你们现在研究 `xuan`，本质上就是在验证：

- 他到底是不是这种 `completion-first inventory manager`

到目前为止，答案是：

- 很大概率是

### 5.4 什么是 `MERGE` / `REDEEM` 在库存管理中的作用

在 Polymarket 二元市场中：

- 如果你同时持有可配对的 `YES` 和 `NO`
- 可以通过 `MERGE` 变回 `$1`

如果市场结束后你持有赢家腿：

- 可以 `REDEEM`

这意味着：

- `MERGE` 是库存压平与现金化
- `REDEEM` 是赢家残留的结算兑现

所以 `xuan` 的利润结构，不只是“低买高卖”，而是：

1. 开仓时获得较好的 inventory geometry
2. 配对后通过 `MERGE` 回收资本
3. 剩余赢家腿再 `REDEEM`

### 5.5 这一章学完，你要得到什么能力

你至少要能解释：

1. 为什么策略的核心对象不是价格，而是库存状态
2. 为什么 `completion-first` 比“继续摊低成本”等待配对更像成熟做市逻辑
3. 为什么 `MERGE` 不是附属动作，而是库存管理核心动作

## 6. 第三章：执行质量与成交概率

这是你们当前最缺、也最容易被忽略的一章。

### 6.1 研究对手，不等于研究可执行性

公开研究可以告诉你：

- `xuan` 在什么状态进场
- 这些状态后面更容易出现什么结果

但公开研究不能自动告诉你：

- 你在同样状态下挂同样的单，能否得到同样成交

这就是：

- `research truth`
- `execution truth`

的差别。

### 6.2 什么是执行质量

对你们来说，执行质量至少包括：

1. 是否成交
2. 多久成交
3. 先成交哪一边
4. 是否只成交了最危险的一边
5. opposite 是否能在目标时间内补齐
6. 最终 repair 花了多少代价

如果只看：

- “有没有赚钱”

你会完全看不清策略的真实问题。

### 6.3 你们当前最该盯的执行指标

对 `pair_arb / PGT`，我建议固定看这些指标：

- `first_leg_fill_rate`
- `opposite_fill_rate_30s`
- `median_opposite_delay_s`
- `same_side_before_opposite_ratio`
- `clean_close_ratio`
- `repair_rate`
- `repair_cost_per_episode`
- `residual_qty_p90`

这套指标的重要性高于：

- 单看总 PnL

### 6.4 为什么 `xuan` 看起来像“30s 内几乎必配”

当前最可能的解释，不是：

- 他有神秘固定 pair target

而是：

1. 他先做了很强的 `open gate`
2. 他进入的是对 opposite fill 更友好的状态
3. 他在不对的状态根本不玩
4. 他对没按预期完成的 episode 有更果断的 repair 边界

换句话说：

- “30s 高成功率”不是单点技巧
- 而是整个状态筛选系统的结果

### 6.5 这一章学完，你要得到什么能力

你至少要能回答：

1. 为什么“首腿能成”不代表策略强
2. 为什么公开对手研究和我方执行真值必须分开
3. 为什么 `xuan` 的 edge 很可能先体现在 `open gate`，再体现在 fillability

## 7. 第四章：事件驱动回测

这一章是把研究变成工程的桥。

### 7.1 为什么不能只做 bar 回测

你们的问题不是：

- 某一分钟涨跌了多少

而是：

- first leg 何时开
- opposite 何时来
- 中间是否出现 same-side add
- 何时 `MERGE`
- 是否 `REDEEM`
- 最后残什么仓

所以如果回测不是 **事件驱动** 的，你看不到最关键的东西。

### 7.2 什么是 episode

你们后续所有研究，都建议围绕 `episode` 展开。

一个 episode 可以定义为：

1. 第一腿开仓
2. opposite fill 逐步发生
3. 中间可能出现 additional fills
4. episode 结束于：
   - clean pair close
   - merge
   - redeem
   - repair
   - 或残仓冻结

### 7.3 一个策略回测最少要重建什么

对你们当前项目，最少要重建：

- `first_leg_ts`
- `first_leg_side`
- `first_leg_price`
- `first_leg_size`
- `first_opposite_ts`
- `first_opposite_delay`
- `same_side_before_opposite`
- `episode_close_ts`
- `episode_close_reason`
- `merge_qty`
- `redeem_qty`
- `repair_cost`
- `final_residual`

### 7.4 为什么 `episode` 比“总交易列表”更重要

因为策略不是靠单笔交易赚钱，而是靠 episode 结构赚钱。

同样两笔成交：

- 在一个 episode 里可能是漂亮配对
- 在另一个 episode 里可能只是灾难中途的一小段

如果你不按 episode 看，就会：

- 误把局部漂亮成交当成好策略

### 7.5 这一章学完，你要得到什么能力

你至少要能自己手工重建一个 episode，并回答：

1. 这个 episode 是怎么开始的
2. 为什么它成功或失败
3. 失败是输在 `open gate`、`fillability` 还是 `repair`

## 8. 第五章：统计证伪与研究口径

这章最容易被忽视，但实际上最关键。

因为你们现在最大的问题之一，不是没有结论，而是：

- 容易过早相信结论

### 8.1 什么是“观察事实”

例如：

- `MERGE` 前 winner-heavier 的比例约 `75.1%`
- `W_first` 的 opposite fill 比 `L_first` 慢
- 某些 bucket 的 `hit30` 更高

这些都属于：

- **观察事实**

### 8.2 什么是“解释”

例如：

- “他有方向性 alpha”
- “他是强 maker geometry”
- “他在挑对侧流动性更好的状态”

这些都属于：

- **解释**

你必须始终把这两层分开。

### 8.3 什么是“实现含义”

例如：

- “那就把高价侧作为默认 first leg”
- “那就把 0.6-0.7 作为默认开仓带”
- “那就允许更晚开仓”

这些属于第三层：

- **实现含义**

最危险的错误就是：

- 从观察直接跳到实现

中间没把解释层做证伪。

### 8.4 你们最常见的 4 个研究错误

#### 错误 1：窗口幻觉

某个现象在 2 天内很强，不代表长期有效。

#### 错误 2：幸存者偏差

你只看见了成功轮次，没看见根本没参与的轮次。

#### 错误 3：口径混算

例如把：

- 短窗口 trades
- lifecycle positions snapshot
- activity history

混在一起下单一结论。

#### 错误 4：拿漂亮叙事替代硬证据

例如：

- “他一定是这样想的”
- “他肯定是挂了什么特殊单”

这些话如果没有数据支撑，最多只是待验证假设。

### 8.5 这一章学完，你要得到什么能力

你至少要能把任何一句研究结论拆成：

1. 观察事实是什么
2. 解释是什么
3. 需要什么新数据来支持或推翻
4. 推翻前为什么不能写进策略

## 9. 如何把这 5 章连起来理解 `xuan`

现在把前面所有内容串起来。

当前最合理的 `xuan` 工作性画像是：

1. 他不是纯方向交易员
2. 他也不是纯中性机械 pair trader
3. 他更像一个：
   - 挑状态开 first leg
   - 追求较高 opposite fillability
   - 又不完全放弃 winner-side quality
   - 并通过 `MERGE/REDEEM` 回收资本和兑现利润
   的库存管理型做市者

所以你研究他的正确顺序应该是：

1. `Open Gate`
2. `Completion`
3. `Inventory / Merge / Redeem`
4. `Repair / Cooldown`

而不是一上来就研究：

- 固定 pair target
- 固定价差公式

## 10. 你现在应该怎么学

下面不是抽象建议，而是你可以直接执行的学习方法。

### 10.1 每次只学一个问题

例如今天只学：

- 为什么 `winner-bias` 和 `30s completion` 不是一个目标

不要一次学太多概念。

### 10.2 每次都用你们自己的数据验证

你不需要先去看很多外部资料。

最好的教材就是：

- 你们自己的 replay
- 你们自己的失败 episode
- `xuan` 的公开行为数据

### 10.3 每学一个概念，都写成你自己的语言

例如：

- 什么是 inventory risk
- 什么是 adverse selection
- 什么是 completion-first

如果你不能用你自己的话解释，就说明你还没真正理解。

### 10.4 永远强制自己写“反例”

比如你得到一个结论：

- “高价侧更像 winner-quality”

你立刻要补一行：

- “什么数据会推翻它？”

这能防止你过早相信故事。

## 11. 现在立刻开始的 7 个练习

这 7 个练习比继续看外部材料更重要。

### 练习 1：手工拆一轮 `xuan` 的成功 episode

写出：

- first leg 何时发生
- opposite 何时出现
- 中间有没有 same-side add
- 何时 `MERGE`
- 最后有没有 `REDEEM`

### 练习 2：手工拆一轮你们自己的失败 episode

写出：

- 为什么 first leg 被允许开
- 为什么 opposite 没按时来
- 为什么后续处理失败

### 练习 3：把一个结论拆成三层

例如：

- 观察：`MERGE` 前 winner-heavier 比率高
- 解释：可能存在 weak first-leg bias + fill geometry amplification
- 实现含义：暂时不能直接写进 live 规则

### 练习 4：写出你们策略当前最重要的 10 个指标

不能只有 PnL。

至少要包括：

- `opposite_fill_rate_30s`
- `median_opposite_delay`
- `same_side_before_opposite_ratio`
- `repair_rate`

### 练习 5：对同一批样本分开算 2 个目标

分别算：

- `winner-quality`
- `completion-quality`

然后比较它们是否冲突。

### 练习 6：给每个研究结论写“证伪条件”

例如：

- 如果更高精度数据里方向翻转，这个结论失效

### 练习 7：给每个策略模块写一句最核心问题

例如：

- `Open Gate`：什么状态值得开第一腿
- `Completion`：什么条件下 opposite 高概率在 30s 内来
- `Repair`：什么时候继续等是错的

4. 你能自己指出一个结论是观察、解释还是实现含义

6. 你能和 agent 讨论时，明确指出“这里证据还不够”

只要做到这 6 条，你就已经不是“完全依赖 agent 的人”了。
